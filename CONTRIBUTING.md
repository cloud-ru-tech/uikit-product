# Styling

## Создание новой темы

1. Копируем существующую тему в папке `theme` (например `purple.ts`).
2. Изменяем название файла. По данному названию будет происходить import темы.
3. Переопределяем необходимые цвета

## Добавление нового цвета

1. Создаем новую css-переменную в файле: `./theme/color/vars`
   Правила наименование экспортируемой переменной
   : `COLORS_${componentName}`
   Например: `COLORS_BUTTON`

   Правила наименования свойств `COLORS_${componentName}`:
   : `${определяющий признак 1}_${определяющий признак 2}_${css-свойство(например, color)}`
   Например: `TRANSPARENT_ACTIVE_COLOR`

   Правила наименовани css-переменной
   : `--${перечесление через "-" предыдущих параметров }`
   Например: `--button-transperent-active-color`

   Полный пример

   ```
   export const COLORS_BUTTON = {
       TRANSPARENT_ACTIVE_COLOR: `--button-transperent-active-color`
   }
   ```

2. Определяем ее в необходимой теме

Например
: `${COLORS_BUTTON.TRANSPARENT_COLOR}: var(${COLORS.VIOLET_3});`

## Правила 
* Пропсы описываем через интерфейсы
* Каждая компонента должна содержать в себе файл ComponentName.stories.tsx, который содержит компоненту `Template`. Из stories.tsx должен экспортироваться настроенный Template, название которого уже в camelCase. (пример компонента - Divider)
* Каждый компонент должен экспортировать себя и свой интерфейс пропсов. Все остальные экспорты уже настраиваются индивидуально.
* Каждая папка компоненты должна содержать в себе `index.ts`, `ComponentName.tsx`, `ComponentName.stories.tsx` (если компонент должен попасть в storybook), `styled` (если компоненту нужна стилизация). Каждая папка компонента в storybook должна содержать в себе `index.ts` (для реэкспорта компонент), папку `components`, которая будет содержать в себе компоненты для storybook, папку `helpers` (для вспомогательных файлов), папку `helperComponents` (для вспомогательных компонент).
